Penalty Box: A simple restful rate limiter

Penalty box exposes an API that lets multiple service use it as a restful rate limiter. That API is documented here.

You may also be interested in the GitHub repository

/health

Health check endpoint

/health get

get

Returns 200 if the service is available

get /health

Returns 200 if the service is available

  • Response

HTTP status code 200

The Penalty Box service is available and serving requests

/rate-limit

Checks whether a request is allowed and keeps track of how often a key has been accessed

/rate-limit post

post

Post info on a request to see if request is allowed. If allowed Penalty Box appends keep track of it to enforce rate limits

post /rate-limit

Post info on a request to see if request is allowed. If allowed Penalty Box appends keep track of it to enforce rate limits

  • Request
  • Response

Query Parameters

  • appName: required (string)

    The name of the application using the rate limiter

  • key: required (string)

    The key the application would like to rate limit

  • cost: required (integer)

    How much this request should cost. Typically each request is cost 1, but this enables weighting different requests differently

  • rateLimit: required (integer)

    How many requests per hour this key should get. This is used in initial setup of the rateLimit. Once written it will not be overwritten

HTTP status code 200

The request was allowed

Headers

  • X-Rate-Limit-Limit: (string)

    The rate limit for this key in this app

  • X-Rate-Limit-Remaining: (string)

    The number of requests remaining to this key for this app

HTTP status code 400

The request was missing one more more required arguments

HTTP status code 403

The request was denied because the rate limit has been reached

Headers
